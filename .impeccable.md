# VeteranPCS — Impeccable Design Context

This file captures the design intent for VeteranPCS so future sessions can make consistent decisions without re-asking. Update in place; do not duplicate sections.

## Design Context

### Users

**Primary:** Active-duty service members, veterans, and military spouses navigating a PCS (Permanent Change of Station) move. They arrive stressed, time-boxed, and often relocating to a state they don't yet live in. Their job-to-be-done is **"find a vetted, military-experienced real estate agent or VA-loan lender for the state I'm moving to — fast, free, and from someone who actually understands BAH, VA loans, and deployment timing."**

**Context of use:** Often mid-move. Frequently mobile. Often during off-hours. Reading level target: 5th–7th grade. May have combat-related vision, hearing, or motor accommodations.

**Secondary:** First-time military homebuyers, agents/lenders applying to join the network, military spouses exploring careers.

### Brand Personality

Three words: **Trusted. Patriotic. Helpful.**

Voice: friendly and conversational, like advice from a military friend who's done this move four times. Plain language. **No emoji. No excessive symbols. No exclamation-stacking.** Military terminology is fine when used correctly and explained on first use.

Slogan and emotional anchor: *"Together, we'll make it home."*

Emotional goals, in priority order:
1. **Calm reassurance** — "we've got you." The user should exhale on landing.
2. **Confidence and competence** — the network is real, vetted, and large (370+ agents across all 50 states). Proof beats promises.
3. **Quiet patriotism** — built by veterans for veterans, not loud flag-waving.

### Aesthetic Direction

The visual language is set by the in-house designer and approved by the CEO and head designer. **Stick with the current designs.** Refinement is welcome; wholesale redirection is not.

**Locked brand tokens (do not change):**
- Primary deep navy: `#292F6C` (with `hover: #4E5489`, `light: #9599b3`)
- Accent deep red: `#a81f23` (with `dark: #7e1618`, `light: #d22230`)
- Accent blue: `#003486`, light blue `#2ea3f2`
- The navy + deep red pairing is the brand. Do not introduce competing accent colors without explicit approval.

**Typography (already configured in `tailwind.config.ts` and `app/layout.tsx`):**
- Inter — default body and UI
- Poppins — marketing copy, often `tracking-wider`
- Roboto — utility / forms
- Lora — editorial headlines and long-form content
- Tahoma (TTF in `/styles`) — legacy hero treatments
- When in doubt, default to Inter for UI and Lora for editorial moments.

**Theme:** Light mode only. Dark mode is **not** on the roadmap.

**Anti-references — what this is NOT:**
- Not a Zillow / Redfin / Compass clone. We are a **referral network**, not a listings marketplace. Do not import generic real-estate UI patterns (map-with-pins-of-properties, infinite-scroll listing grids, "see 1,247 homes" badges).
- Not stock-photo flag-waving. No oversaturated American-flag overlays, no salute-at-sunset stock imagery.
- Not a fintech-startup makeover. The brand is patriotic and warm, not Stripe-minimal.

### Design Principles

These are the rules of thumb for every design decision in this codebase.

1. **Clarity beats cleverness — shorten the path to an agent.**
   The current path from landing to "I have an agent's name in my state" is too long. Every page, section, and component should be evaluated against: *does this move the user closer to a vetted agent, or further?* If a section doesn't earn its place against that yardstick, cut it or demote it. The home page's primary CTA is **Find An Agent** — treat it as the load-bearing pixel.

2. **The network is the product. Lead with humans.**
   Surface real agents, real lenders, real military credentials, real reviews, real charity impact. Photos of people > illustrations of houses. Specific numbers (370+ agents, 50 states, $4,000 max cash back, 80+ five-star reviews) > vague adjectives. Do not bury the network behind generic real-estate chrome.

3. **Calm authority, not sales pressure.**
   Tone reads like a trusted friend, not a quota-driven funnel. No urgency timers, no "limited spots," no exclamation-stacking. Plain language at 5th–7th grade reading level. Military terms are explained when first used. Where you would be tempted to use an emoji, use a clean inline icon from `/public/icon` instead.

4. **Honor the locked brand; refine within it.**
   Navy `#292F6C` and red `#a81f23` are non-negotiable. Light mode only. Use existing radius (`32px` `rounded-custom`), existing shadows (`shadow-header`, `shadow-nav`), existing gradient (`bg-gradient-custom`). When you need a new token, extend the Tailwind theme — do not hardcode hex values in components.

5. **Accessibility is the floor — target WCAG 2.1 AA, push to AAA where reasonable.**
   The audience includes service members with combat-related vision, hearing, and motor needs. Color contrast must meet AA at minimum (AAA for body text where reachable). Respect `prefers-reduced-motion` on AOS / slick-carousel animations. Tap targets ≥ 44px. Forms must be keyboard- and screen-reader-friendly with visible labels (no placeholder-as-label). Alt text on every meaningful image; decorative images get `alt=""`.

### How to use this file

When making any design or UX change in this codebase, re-read the **Design Principles** above and verify the change passes all five. If a change requires breaking a principle (e.g., the CEO requests an emoji, a stakeholder wants a new accent color), surface that tension to the user before implementing — don't quietly violate a principle.
